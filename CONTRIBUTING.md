# Contributing

## Before You Start

- Open an issue for bugs/features before large changes.
- Keep changes focused and small when possible.
- Never commit secrets or private credentials.

## Development Setup

```bash
npm install
npm run build
npm test
```

## Pull Request Checklist

- Add or update tests for behavior changes.
- Run `npm run typecheck` and `npm test`.
- Update `README.md` and `CHANGELOG.md` when user-facing behavior changes.
- Keep API/tool behavior backwards compatible when practical.

## Commit Guidance

- Use clear, imperative commit messages.
- One logical change per commit.

## Questions

If something is unclear, open a discussion in issues before implementing large design changes.
