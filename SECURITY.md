# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities privately before opening a public issue.

- Open a GitHub Security Advisory if available.
- If advisories are unavailable, open an issue with minimal public detail and ask maintainers for a private channel.

Include:

- Affected version(s)
- Reproduction steps or proof of concept
- Impact assessment
- Suggested mitigation (if known)

## Response Expectations

- Initial acknowledgment target: within 3 business days
- Status updates target: at least weekly until resolution
- Fix release target: based on severity and reproducibility

## Supported Versions

Only the latest published minor line is actively supported for security fixes.
